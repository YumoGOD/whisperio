# Функциональная спецификация

## Что делает приложение

Веб-сервис, в который пользователь загружает аудио/видео-файл, получает транскрипт.

## Пользовательские сценарии

### Сценарий 1: Загрузка одного файла
1. Пользователь открывает главную страницу
2. Перетаскивает файл в drop-zone (или выбирает через диалог)
3. Опционально выбирает: язык (auto / ru / en / ...), модель (large-v3 / large-v3-turbo / medium), включить ли диаризацию (на v1 — не реализуем, оставить чекбокс задизейбленным)
4. Жмёт «Транскрибировать»
5. Видит карточку задачи со статусом: `queued → preprocessing → transcribing → done` (или `failed`)
6. По завершении видит транскрипт с таймкодами сегментов, кнопки: «Копировать», «Скачать .txt», «Скачать .srt», «Скачать .json»

### Сценарий 2: Просмотр истории
1. На второй вкладке («История») — таблица последних 100 задач
2. Колонки: имя файла, длительность, модель, статус, дата, действия (открыть / удалить)
3. Клик по строке — открывает страницу результата

### Сценарий 3: Массовая загрузка
1. Пользователь выбирает несколько файлов разом (до 50 за раз)
2. Каждый создаётся как отдельная задача
3. Воркер обрабатывает их по одному (один GPU)

## Поддерживаемые форматы

Вход: `.mp3, .wav, .m4a, .ogg, .flac, .mp4, .mov, .mkv, .webm, .avi`
(FFmpeg на лету конвертит всё в 16kHz mono WAV)

Лимит размера файла: настраивается через `.env` (по умолчанию 2 GB).

## Что НЕ входит в v1

- Авторизация и пользователи
- Диаризация (кто говорит) — кнопка есть, но задизейблена
- Стриминговая транскрипция в реальном времени
- Редактирование транскрипта в UI
- Поиск по транскриптам
- Webhooks / уведомления

## Поведение при ошибках

- GPU OOM → задача помечается `failed` с сообщением, воркер продолжает работу
- Кривой файл → `failed` на этапе preprocessing
- Воркер упал → systemd/Docker рестартит, задачи в статусе `processing` старше 1 часа автоматически возвращаются в `queued` при старте воркера

## Качество транскрипции

Pipeline на каждый файл:
1. FFmpeg → 16kHz mono WAV
2. (Опционально, флаг в .env) DeepFilterNet денойз — пока **не реализуем в v1**, оставить заглушку
3. Silero VAD → разбиение на речевые сегменты (макс 30 сек)
4. faster-whisper батч-инференс по сегментам (batch_size из .env)
5. Склейка сегментов с таймкодами
6. Сохранение в БД + файлы (.txt, .srt, .json) на диск
